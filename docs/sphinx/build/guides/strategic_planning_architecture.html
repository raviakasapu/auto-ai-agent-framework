Strategic Planning ArchitectureÔÉÅ

The Problem with Simple RoutingÔÉÅ

Current (Inefficient):ÔÉÅ

User: "Add relationship, then validate"
    ‚Üì
Orchestrator üß†: "It's a powerbi-designer task" ‚Üí route
    ‚Üì
Manager üß†: "It's a schema task" ‚Üí route  
    ‚Üì
Worker üß†: "What exactly do I do?" ‚Üí re-plan everything

Issues:

  • ‚ùå 3 LLM calls just for routing

  • ‚ùå No real planning until worker level

  • ‚ùå Worker reinvents the wheel

  • ‚ùå No context passing

Strategic Planning Architecture (Efficient)ÔÉÅ

With StrategicPlanner:ÔÉÅ

User: "Add relationship between Sales and Customers, then validate"
    ‚Üì
Orchestrator üß† STRATEGIC PLANNING:
  Analysis:
    - Multi-step task: relationship + validation
    - Needs: schema modification + validation
    - Dependencies: Must verify tables exist first
  
  CREATES PLAN:
    Step 1: Verify tables ‚Üí list_tables
    Step 2: Find key columns ‚Üí list_columns(Sales), list_columns(Customers)
    Step 3: Create relationship ‚Üí add_relationship(SalesID, CustomerID)
    Step 4: Validate model ‚Üí validate_model
    Rationale: "Systematic approach ensures clean relationship creation"
    Primary worker: powerbi-designer
    
  DELEGATES TO: pbi_manager WITH PLAN + CONTEXT
    ‚Üì
Manager üß† TACTICAL EXECUTION:
  Receives: Full plan + "This is schema+validation task"
  
  Executes plan steps:
    Step 1: schema worker ‚Üí list_tables (context: "checking existence")
    Step 2: schema worker ‚Üí list_columns x2 (context: "finding keys for relationship")
    Step 3: schema worker ‚Üí add_relationship (context: "connecting Sales to Customers")
    Step 4: validator worker ‚Üí validate (context: "verify relationship integrity")
  
  Synthesizes: "Relationship created, validation passed, star schema intact"
    ‚Üì
Workers: Execute focused actions with clear context
    ‚Üì
User ‚Üê Synthesized, analyzed result

ImplementationÔÉÅ

1. StrategicPlanner (framework/components/planners.py)ÔÉÅ

Purpose: Creates execution plans at director level

class StrategicPlanner(BasePlanner):
    """Creates multi-step execution plans with context."""
    
    def plan(self, task, history):
        # LLM creates strategic plan
        plan = {
            "steps": [
                {"action": "list_tables", "worker": "schema", "context": "..."},
                {"action": "add_relationship(...)", "worker": "schema", "context": "..."}
            ],
            "rationale": "...",
            "primary_worker": "powerbi-designer"
        }
        
        # Return Action with plan embedded
        return Action(
            tool_name="powerbi-designer",  # Primary worker
            tool_args={"strategic_plan": plan, "original_task": task}
        )

2. Enhanced Manager (framework/core/manager.py)ÔÉÅ

Now accepts and uses strategic plans:

async def run(
    self, 
    task: str,
    strategic_plan: Optional[Dict] = None,  # NEW
    context: Optional[str] = None  # NEW
):
    # Add plan to memory for worker context
    if strategic_plan:
        self.memory.add({"type": "strategic_plan", "content": strategic_plan})
    
    # Execute with context
    result = await worker.run(task, strategic_plan=strategic_plan, context=context)

3. ConfigurationÔÉÅ

orchestrator.yaml (with Strategic Planning):ÔÉÅ

spec:
  planner:
    type: StrategicPlanner  # ‚Üê Instead of WorkerRouterPlanner
    config:
      inference_gateway: openai-llm
      worker_keys: [model-analysis, math, powerbi-designer]
      planning_prompt: |
        You are a strategic planner for an AI system.
        
        Analyze the task and create a detailed execution plan:
        1. Break complex tasks into concrete steps
        2. Identify dependencies between steps
        3. Assign each step to appropriate worker
        4. Provide context for each step
        
        Return JSON:
        {
          "plan": {
            "steps": [
              {"action": "list_tables", "worker": "powerbi-designer", "context": "Verify tables exist"},
              {"action": "add_relationship(...)", "worker": "powerbi-designer", "context": "Create link"}
            ],
            "rationale": "Why this plan works",
            "primary_worker": "powerbi-designer"
          }
        }

BenefitsÔÉÅ

‚úÖ Single Planning LLM CallÔÉÅ

  • Orchestrator plans everything upfront

  • No redundant routing calls

  • Workers execute without re-planning

‚úÖ Rich Context PassingÔÉÅ

Worker receives:
- Strategic plan: What needs to be done
- Step context: Why this step matters
- Original task: User's intent
- Manager context: Domain-specific guidance

‚úÖ Better ReasoningÔÉÅ

  • Director sees big picture

  • Manager adds domain expertise

  • Worker focuses on execution

‚úÖ Traceable PlansÔÉÅ

Logs show:
[Orchestrator] Created 4-step plan
[Manager] Executing step 1 of 4: list_tables
[Worker] Completed: list_tables
[Manager] Executing step 2 of 4: add_relationship
...

MigrationÔÉÅ

Simple (Current): WorkerRouterPlannerÔÉÅ

# Simple keyword-based routing
planner:
  type: WorkerRouterPlanner
  config:
    worker_keys: [model-analysis, powerbi-designer]
    rules:
      - worker: powerbi-designer
        include: ["table", "relationship"]

Use when: Simple classification, single-step tasks

Advanced (New): StrategicPlannerÔÉÅ

# Deep planning with context
planner:
  type: StrategicPlanner
  config:
    worker_keys: [model-analysis, powerbi-designer]
    planning_prompt: |
      Create detailed execution plans...

Use when: Complex multi-step tasks, need context passing

Example FlowsÔÉÅ

Example 1: Simple QueryÔÉÅ

"List all tables"
  ‚Üí Strategic Plan: Simple 1-step query
  ‚Üí powerbi-designer ‚Üí schema ‚Üí list_tables
  ‚Üí Done

Example 2: Complex TaskÔÉÅ

"Create relationship between Sales and Products, ensure it follows star schema"
  ‚Üí Strategic Plan:
      1. list_tables ‚Üí verify both exist
      2. list_relationships ‚Üí understand current schema
      3. list_columns(Sales), list_columns(Products) ‚Üí find keys
      4. add_relationship(SalesID ‚Üí ProductID)
      5. validate ‚Üí ensure star schema intact
  ‚Üí powerbi-designer manager executes 5 steps
  ‚Üí Synthesizes: "Relationship created, star schema maintained"
