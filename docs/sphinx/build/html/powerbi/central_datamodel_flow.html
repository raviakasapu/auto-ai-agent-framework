How job_id Flows Through the Systemïƒ

1. Frontend Request (WebSocket)ïƒ

// src/services/agent/websocket.ts

sendMessage({
  task: "Add a measure called TotalRevenue",
  job_id: activeJobId  // â† Automatically injected by hook
});

2. Backend Receives (main.py)ïƒ

# main.py - WebSocket handler
job_id = (message or {}).get("job_id")  # â† Extract from request

# Map job_id to model_id (they're the same)

# Set as environment variable
os.environ["MODEL_ID"] = model_id  # â† Now available globally

print(f"[WebSocket] âœ“ MODEL_ID set to: {model_id} (from job_id)")

3. Tools Auto-Detect model_idïƒ

# framework/components/datamodel_tools.py
def _get_model_id(model_id: Optional[str] = None) -> str:
    # 1. Explicit parameter (if agent provides it)
    if model_id:
        return model_id
    
    # 2. Environment variable (set by main.py) âœ… THIS IS WHAT HAPPENS
    env_model_id = os.getenv("MODEL_ID")
    if env_model_id:
        return env_model_id
    
    # 3. Fallback to first model
    service = get_datamodel_service()
    models = service.get_all_models()
    if models:
        return models[0]
    
    raise ValueError("No model_id available")

4. Agent Calls Toolïƒ

# Agent doesn't need to know about model_id!
# It just calls the tool naturally:
tool.execute(table_name="Sales", measure_name="TotalRevenue", expression="SUM(...)")

# Tool automatically gets model_id from environment âœ…

5. DataModelService Operationsïƒ

# framework/services/datamodel_service.py
def add_measure(self, model_id: str, table_name: str, measure_data: dict):
    # Operates on datamodel.json
    model = self._cache["models"][model_id]  # â† Uses job_id
    table = find_table(model, table_name)
    table["measures"].append(measure_data)
    self._save_datamodel()  # â† Writes to file

6. Sync Back to Backendïƒ

# main.py - after agent completes
sync = get_api_sync()
if sync.is_enabled():
    sync.push_to_backend()  # â† Sends updated datamodel.json to your backend API

Key Pointsïƒ

âœ… Frontend sends job_id - Already implemented in useAgentWebSocket

âœ… Backend receives and maps - job_id â†’ model_id â†’ os.environ["MODEL_ID"]

âœ… Tools auto-detect - No need for agent to parse or provide model_id

âœ… Universal across platforms - Works for Power BI, Tableau, Looker, etc.

âœ… Bidirectional sync - Backend API â†” datamodel.json â†” Agents
