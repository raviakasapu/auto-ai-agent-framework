Getting Started (Complete Guide)
==================================

Welcome to the AI Agent Framework! This guide will get you from zero to a working agent in 10 minutes.

Prerequisites
-------------

Before you begin, make sure you have:

* Python 3.11 or higher
* pip (Python package installer)
* A terminal/command prompt
* (Optional) A code editor like VS Code

Quick Check
~~~~~~~~~~~

Open your terminal and verify Python is installed::

    python3 --version

You should see something like ``Python 3.11.x`` or higher.

Installation
------------

Step 1: Clone or Navigate to the Repository
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Clone the repo (or pull the latest changes) and open it::

    git clone https://github.com/akasa-ai/AI_Agent_Framework.git
    cd AI_Agent_Framework

Step 2: Create Virtual Environment
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Create an isolated Python environment for the project::

    python3 -m venv venv

This creates a ``venv/`` directory with a clean Python installation.

Step 3: Activate Virtual Environment
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**On macOS/Linux**::

    source venv/bin/activate

**On Windows**::

    venv\Scripts\activate

You should see ``(venv)`` appear in your terminal prompt.

Step 4: Install Dependencies
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Install the reusable library plus repo tooling::

    pip install -e ./agent-framework-pypi
    pip install -r requirements.txt

The editable install exposes the ``agent_framework`` package locally while
``requirements.txt`` covers FastAPI, Uvicorn, Pydantic, Sphinx, and tooling for the docs server.

Verify Installation
-------------------

Test that everything is working::

    python -c "from deployment.factory import AgentFactory; print('âœ“ Installation successful!')"

If you see ``âœ“ Installation successful!``, you're ready to go!

First Steps
-----------

Now that you're set up, here's what to do next:

**Option A: Run a Demo (Fastest)**

Run the existing research assistant agent::

    python tests/quick_run.py

You'll see the agent execute and print results.

**Option B: Start the Documentation Server**

Launch the interactive documentation::

    AGENT_MANIFEST_PATH=docs/agent_manifest.json uvicorn docs_server.main:app --reload --port 8002

Then open http://127.0.0.1:8002 in your browser.

**Option C: Create Your First Agent**

Jump straight into the tutorial: :doc:`../tutorials/01_new_agent`

Key Concepts
------------

Before diving deeper, understand these core concepts:

**1. Agent**
   An autonomous entity that can use tools to accomplish tasks.
   
**2. Tool**
   A specific capability (e.g., search, calculate, read file).
   
**3. Planner**
   Decides which tool to use and when. Can be:
   
   * **StaticPlanner** - Always uses the same tool
   * **LLMRouterPlanner** - Uses LLM to decide based on user input
   
**4. Gateway**
   Interface to LLMs (OpenAI, Anthropic, or mock for testing).
   
**5. Memory**
   Stores conversation history and context.

Example: How It All Fits Together
----------------------------------

Here's a simple example of how components work together:

.. code-block:: python

    from deployment.factory import AgentFactory
    import asyncio

    async def main():
        # Load agent from YAML configuration
        agent = AgentFactory.create_from_yaml(
            "configs/agents/research_assistant.yaml"
        )
        
        # Run the agent with a task
        result = await agent.run(
            query="What is Python?",
            region="us-en"
        )
        
        print(f"Result: {result}")

    # Run the async function
    asyncio.run(main())

When you run this:

1. âœ… **Factory** reads the YAML config
2. âœ… **Resources** are instantiated (tools, LLM gateway, memory)
3. âœ… **Agent** is assembled with planner and tools
4. âœ… **Planner** decides which tool to use
5. âœ… **Tool** executes and returns results
6. âœ… **Memory** stores the interaction
7. âœ… **Result** is returned to you

Troubleshooting
---------------

**Problem: "ModuleNotFoundError"**

*Solution:* Make sure virtual environment is activated::

    source venv/bin/activate  # macOS/Linux
    venv\Scripts\activate     # Windows

**Problem: "No such file or directory: requirements.txt"**

*Solution:* Make sure you're in the repository root directory::

    cd /path/to/sdk
    pwd  # Should end with "/sdk"

**Problem: "Command 'python3' not found"**

*Solution:* Try ``python`` instead of ``python3``, or install Python from python.org

**Problem: "Permission denied"**

*Solution:* On macOS/Linux, you might need to make scripts executable::

    chmod +x tests/quick_run.py

**Problem: Agent fails with "MODEL_DIR not set"**

*Solution:* Some agents need a model directory. Set it::

    export MODEL_DIR=/path/to/your/Model
